# CLAUDE.md

## Project Overview

Private Model Serving — confidential model serving built on **Shade** (CVM framework) and **Atlas** (attested TLS).

Model owner pushes weights into a CVM, end users verify integrity via aTLS, then run inference — without trusting the cloud provider.

## Repository Structure

```
private-model-serving/
├── app/                        # model-service (FastAPI): /push-model, /model-hash, /health, /ready
├── scenarios/model-owner/      # Model owner simulation (hash, push token, tests)
├── scenarios/user/             # End user simulation (expected hash, tests)
├── scripts/encrypt_env.py      # Phala env encryption (x25519 + AES-256-GCM)
├── docker-compose.yml          # App services (vLLM + model-service)
├── shade.yml                   # Shade config (routes, domain, plugins)
├── docker-compose.shade.yml    # Generated by `shade build` (DO NOT EDIT)
├── policy.json                 # Generated by `make atls-policy` (DO NOT EDIT)
├── Makefile                    # All targets
├── .env                        # PUSH_TOKEN (not committed)
└── CLAUDE.md
```

## Commands

```bash
# Local dev (no TLS, no attestation)
make app-dev-up         # Start vllm + model-service
make app-dev-test       # Push model + run tests
make app-dev-down

# Phala deployment
make shade-build        # Generate docker-compose.shade.yml
make phala-deploy       # Deploy new CVM
make phala-update       # Update compose + restart existing CVM
make phala-status       # Check CVM status
make phala-logs         # View logs

# aTLS scenario (attested TLS via Atlas)
make atls-setup              # Build atlas Python binding (once)
make atls-policy             # Generate policy.json from live CVM
make atls-model-owner-push   # Push model via aTLS
make atls-remote-hash        # Get model hash via aTLS
make atls-hash-verify        # Compare local vs remote hash
make atls-user-infer         # Run inference via aTLS

# Tests
cd app && uv run pytest tests/ -v
```

## Key Concepts

- **shade.yml** declares routes, domain, TLS config. `shade build` merges it with docker-compose.yml.
- **policy.json** contains TDX bootchain measurements, OS image hash, and app compose. Generated from live CVM via `shade policy generate`.
- **Model push**: one-time POST of tar archive + Bearer token to `/push-model`. 410 Gone on repeat.
- **Model hash**: deterministic SHA-256 of all model files. Same algorithm on model-owner side and CVM side.
- **EKM session binding**: `report_data = SHA512(nonce + EKM)` — binds attestation to TLS session (RFC 9266).

## Code Style

- Python 3.11+, managed with `uv`
- Ruff: line-length=100, 4-space indent, double quotes, Google docstrings
- Conventional Commits: `feat(app): ...`, `fix(shade): ...`

## Safety

- Never commit secrets; `.env` and `push-token.txt` are in `.gitignore`
- Treat attestation, TLS, EKM, and certificate flows as sensitive
- `policy.json` contains CVM measurements — do not commit
